Combo Box

A custom select element that turns into a popover select for mobile screens.

Custom Element
  • HTML: <combo-box>
  • JS: ComboBox extends OpenableElement

How it works​

There are two pieces to this component: a <button> styled like a select element which will open the <combo-box> which is our custom styled choices.

How to use​

Full Example​

<!-- An example using a Shopify product form --><div class="select-wrapper">  <combo-box     initial-focus-selector="[aria-selected='true']"     id="{{ option_id }}-combo-box"     class="combo-box">    <span class="combo-box__overlay"></span>    <!-- Header shown on popover for mobile screens -->    <header class="combo-box__header">      <p class="combo-box__title">{{ option.name }}</p>      <button         type="button"         class="combo-box__close-button tap-area"         data-action="close"         title="{{ 'general.accessibility.close' | t | escape }}">        {%- render 'icon' with 'close' -%}      </button>    </header>    <!-- Option list -->    <div class="combo-box__option-list" role="listbox">      {%- for value in option.values -%}        <button           type="button"           role="option"           class="combo-box__option-item"           value="{{ value | escape }}"           aria-selected="{% if value == option.selected_value %}true{% else %}false{% endif %}">{{ value }}</button>      {%- endfor -%}    </div>    <!-- Hidden select element which we bind to -->    <select       class="visually-hidden"       name="option{{ option.position }}"       form="{{ product_form_id }}"       data-bind-value="{{ option_id }}-value"       aria-label="{{ option.name | escape }}">      {%- for value in option.values -%}        <option value="{{ value | escape }}" {% if value == option.selected_value %}selected{% endif %}>{{ value }}</option>      {%- endfor -%}    </select>  </combo-box>  <!-- The "custom" select element that users will see -->  <button     type="button"     is="toggle-button"     class="select"     aria-expanded="false"     aria-haspopup="listbox"     aria-controls="{{ option_id }}-combo-box">    <span id="{{ option_id }}-value" class="select__selected-value">{{ option.selected_value }}</span>    {%- render 'icon' with 'chevron' -%}  </button></div>

Button Element​

<!-- An example using a Shopify product form --><div class="select-wrapper">  <combo-box>    [ content here ]  </combo-box>  <button     type="button"     is="toggle-button"     class="select"     aria-expanded="false"     aria-haspopup="listbox"     aria-controls="{{ option_id }}-combo-box">    <span id="{{ option_id }}-value" class="select__selected-value">{{ option.selected_value }}</span>    {%- render 'icon' with 'chevron' -%}  </button></div>
  • The <button> element has:
    • .select class so it is styled like a select element
    • is="toggle-button" attribute so it takes behaviour from a ToggleButton web component in our JS file.
    • aria-expanded attribute set to false
    • aria-haspopup="listbox" indicating what the element is from an a11y perspective
    • aria-controls which matches the id of the <combo-box>
      • a <span> element inside it with an id that will match the data-bind-value of our hidden select element in the <combo-box>

Combo Box​

<!-- An example using a Shopify product form --><div class="select-wrapper">  <combo-box     initial-focus-selector="[aria-selected='true']"     id="{{ option_id }}-combo-box"     class="combo-box">    [ Combo Box Content ]  </combo-box>  <!-- The "custom" select element that users will see -->  <button></button></div>
  • The <combo-box> has:
    • .combo-box class
    • initial-focus-selector which utilises the tabbable and focus-within extensions to set what element should be get focus when opened.
    • id which matches the aria-controls attribute of the outside <button> element

Combo Box content​

<!-- An example using a Shopify product form -->...    <span class="combo-box__overlay"></span>    <!-- Header shown on popover for mobile screens -->    <header class="combo-box__header">      <p class="combo-box__title">{{ option.name }}</p>      <button         type="button"         class="combo-box__close-button tap-area"         data-action="close"         title="{{ 'general.accessibility.close' | t | escape }}">        {%- render 'icon' with 'close' -%}      </button>    </header>    <!-- Option list -->    <div class="combo-box__option-list" role="listbox">      {%- for value in option.values -%}        <button           type="button"           role="option"           class="combo-box__option-item"           value="{{ value | escape }}"           aria-selected="{% if value == option.selected_value %}true{% else %}false{% endif %}">{{ value }}</button>      {%- endfor -%}    </div>    <!-- Hidden select element which we bind to -->    <select       class="visually-hidden"       name="option{{ option.position }}"       form="{{ product_form_id }}"       data-bind-value="{{ option_id }}-value"       aria-label="{{ option.name | escape }}">      {%- for value in option.values -%}        <option value="{{ value | escape }}" {% if value == option.selected_value %}selected{% endif %}>{{ value }}</option>      {%- endfor -%}    </select>    ...</div>
  • The <combo-box> has the following child elements:
    • <header> element which shows on mobile popover with a title and close button.
    • .combo-box__option-list elmeent which lists all available values. Each option is stored in a <button>. The following are important attributes:
      • .combo-box__option-item class
      • aria-selected attribute which will set true or false if it is selected
    • The hidden <select> element which controls the actual data. The following are important attributes
      • data-bind-value attribute which matches the outer button > span#id

Where it's used​

  • Product Form: We use this for any dropdown selectors on the product form
